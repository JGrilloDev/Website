Networking: Chat Room Client

In the semester term of my Second year of University, I chose the Distributed Systems Module. I made this choice as it was the most confusing, but most interesting topic from when we did our "taster sessions" in the first semester. I started out wanting to know more, and ended with a serious passion for this niche.
Our first task in this module was to create a Multi-threaded application in Python to act as a Client for a chat room. The threads were used to handle user inputs, such as sending a message or choosing another client to message privately, and recieving information from the server.
We used sockets to faciliate commmunication between client and server, and this was my first look at how information was sent. In this case, the information moving between my client and the server consisted of two parts:

  1. A number, of fixed length 2 bites, which contains the length of the incoming dictionary when encrypted
  2. A Dictionary, of variable length, containing various pieces of information

The length of the incoming dictionary has to be disclosed ahead of time, as is a minor drawback of socket commnication. The Dictionary in most cases contained two keys, ID and Message. When a client or the server recieved a message, they would act depending on the ID recieved, and use the other values within the dictionary as parameters. For example, A client can send a message with an ID of 2 and Message being their name, the server will see the ID of two and react by changing the name of the Client to the value of their Message. The server will then inform all clients of the name change.

Global Game Jam 2020

"The valiant cursor must destroy the virus, but its only ability is to repair corrupted folders that act as weapons.
A bullet hell-esque game where the player has no direct means of attacking the enemy. Instead, they must focus on constantly repairing the turrets to attack the virus for them."
The theme of this years Global Game Jam was "Repair". I was working in a team of just myself and another programmer, Paul. The majority of programming we did on this project was as a pair, as we have different pockets of knowledge.
When the game had been built up enough that we were happily able to work on different areas, my focus mainly on building what we needed for UI, such as buttons and the functionality to move between scenes.
This was my first ever Game Jam, and I can happily say it was a very enjoyable experience, as well as very rewarding for being able to create a well rounded game in such a short amount of time.
